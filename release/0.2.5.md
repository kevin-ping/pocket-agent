# Release 0.2.5 — Bridge API

## 新功能

### PA Bridge API (`POST /bridge/send`)

第三方 app 现在可以通过 PA 桥接 Hermes，无需直接连接 Hermes。

**架构链路**：
```
第三方 app → PA /bridge/send → Hermes 流式返回 → PA push_to_self → TTS 播报 → 用户
```

**核心设计**：
- PA = 前台交互壳层（thinking 动画、TTS 播报、展示文字）
- Hermes = 唯一大脑（思考、调业务 API、决定回复内容）

**接口参数**：

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| source | string | ✅ | 来源 app 标识（如 "chess-app"） |
| session_id | string | ✅ | 来源 app 的会话 ID |
| text | string | ✅ | 发送给 Hermes 的消息 |
| user_language | string | ❌ | 用户语言偏好 |
| context | string | ❌ | 附加上下文 |
| show_thinking | bool | ❌ | 是否显示 thinking 动画（默认 true） |

**Session 隔离**：`bridge:{source}:{session_id}`
- Bridge session 和 PA 日常聊天完全隔离
- 不同 source app 之间互不干扰

## 架构改动

### 双入口 + 共享 Hermes dispatch core

```
send_message()          — UI 入口（语音/文字聊天）
  ↓
run_hermes_turn()       — 共享调度内核
  ↑
dispatch_bridge_message() — Bridge 入口（第三方 app）
```

**共享内核** `run_hermes_turn()` 负责：
- SSE 流式消费
- Reasoning / ToolCall 事件按 mode 分流
- 重试逻辑

**UI 入口额外处理**：daily summary、语言 suffix、聊天历史、TTS
**Bridge 入口额外处理**：session 隔离、thinking fallback、push_to_self 播报

### 事件隔离

| Mode | Thinking 事件 | Tool Call 事件 |
|------|--------------|---------------|
| UI | chat-thinking | chat-tool-call |
| Bridge | bridge-thinking | bridge-tool-call |

### Thinking Fallback

Bridge 模式覆盖四种退出条件：
- `/push` 到来 → 完整 UI 收尾
- 回复为空 → 退出 thinking
- 流报错 → 显示错误
- 30 秒超时 → 自动退出
