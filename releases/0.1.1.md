# Pocket Agent v0.1.1

**Release Date**: 2026-05-03

Push notification API and UI polish.

## New Features

### Local Push API
- **HTTP API server** — embedded axum server on `localhost:8650`, auto-starts with the app
- **POST /push** — receive push messages from external sources (Hermes cron jobs, scripts, etc.)
  - `text` (required) — message content to speak and display
  - `emotion` (optional) — tone for display speed: `friendly | cheerful | calm | serious | sad | whisper | excited | angry`
  - `voice` (optional) — override auto-detected TTS voice (e.g. `zh-CN-XiaoxiaoNeural`)
- **Bearer token auth** — uses the same `API_SERVER_KEY` from `.env` as Hermes gateway. Skipped if key is empty.
- **GET /health** — health check endpoint
- **CORS enabled** — permissive for localhost development

### Direct TTS Command
- **`speak_text`** — new Tauri command for playing text as speech without going through LLM. Powers the push API flow.

## Improvements

### Hotkey
- **300ms grace period** on hotkey capture — prevents accidental registration from key-up bounce

### Accessibility
- **Reliable permission check** — replaced osascript detection with direct CGEventTap creation test
- **Auto-open System Settings** — when accessibility permission is missing, automatically navigates to the correct preference pane
- **Error propagation** — capture flow properly reports permission failures instead of silently exiting

### UI
- **SVG icon system** — new `Icon.svelte` component (Lucide style) replacing emoji throughout the UI
- **Icons**: message-square, settings, trash-2, bell, bell-off, x, alert-triangle
- **Avatar zoom fix** — hover scale now applies only to the avatar circle, not the corner frame

## API Example

```bash
curl -X POST http://localhost:8650/push   -H "Authorization: Bearer YOUR_API_KEY"   -H "Content-Type: application/json"   -d '{"text":"Morning report ready!","emotion":"cheerful"}'
```

## Internal
- Added `axum 0.8` and `tower-http 0.6` dependencies
- `chatStore.addBotMessage()` for displaying push messages in chat
- `api/server.rs` — new module with auth middleware
