# Pocket Agent v0.2.3

**Release Date**: 2026-05-11

Major feature drop — chat history, TTS toggle, double-click recording mode,
hotkey system overhaul, and code review fixes.

## New Features

### Chat History
- **SQLite persistence** — all chat messages saved to local database via `rusqlite`.
- **HTML export viewer** — browse past conversations in a generated HTML page.
- **Markdown/link support** — ChatPanel renders HTML content via `{@html}` with
  sanitizer to prevent XSS.

### TTS Toggle (Voice Output)
- **Settings toggle** — new Voice Output switch in Settings panel to enable/disable
  audio playback independently.
- **TTS-off behavior** — when voice output is disabled, text streaming (typewriter
  effect) and SPEAKING animation still work normally. Only audio pronunciation is
  suppressed.
- **Animation lifecycle** — SPEAKING animation auto-stops when typewriter finishes
  in TTS-off mode (reactive `$:` statement detects `isStreaming` → `false`).

### Double-Click Recording Mode
- **Atomic timestamp detection** in CGEventTap callback — detects double-clicks on
  the hotkey via `AtomicI64` timestamp comparison (< 400ms interval).
- **Settings toggle** — enable/disable in Settings panel. When enabled, single
  hotkey press is ignored; double-press starts/stops recording.

### Modifier Key Hotkey Support
- **RightShift, Shift, Cmd, Option, Ctrl, CapsLock** — all now work as push-to-talk
  hotkeys. macOS generates `kCGEventFlagsChanged` events for modifier keys (not
  `kCGEventKeyDown`). The CGEventTap now listens for both event types.
- **Press-to-toggle behavior** — modifier keys use a state tracker so only the press
  (not release) triggers recording toggle, matching fn key behavior.
- **Capture mode handles modifiers** — when capturing a new hotkey, modifier key
  events are intercepted and properly tracked. No false recording triggers when
  changing hotkeys.

### Smarter Capture Logic
- **fn/globe key** — generates both `KEY_DOWN` (179) and `FLAGS_CHANGED` (63) events.
  Capture now prefers the `KEY_DOWN` code (179 → "fn") over the globe keycode
  (63 → "Key(63)").
- **Multi-capture fix** — `OnceLock` replaced with `Mutex<Option<Sender>>`. Previously
  the second capture call silently failed because `OnceLock::set()` can only be called
  once. New channel created every capture.
- **Default hotkey** — changed from fn (179) to RightShift (60).

### Better Logging
- **Human-readable hotkey names** — `update_hotkey` now returns the display name to
  the frontend. Logs show `[hotkey] hotkey updated: 179 [fn] -> 60 [RightShift]`.
- **Conditional config logs** — `[config] voice_output set to: true/false` and
  `[hotkey] double_click_mode set to: true/false` only print when values actually
  change.

## Bug Fixes

- **TTS toggle save failure** — `settings.ts` referenced undeclared variable
  `oldTtsEnabledValue`, causing `ReferenceError` and "保存失败，请重试" on every save.
  Fixed by declaring the variable in `save()` scope.
- **Settings reset on save** — `settings.ts save()` was merging from `defaults`
  instead of current store state, causing unmodified fields to reset. Fixed to
  `{ ...current, ...partial }`.
- **TTS parameter name mismatch** — Rust `_tts_enabled` (leading underscore) caused
  Tauri 2.0 to generate `_ttsEnabled` instead of `ttsEnabled`, making the frontend
  value always `undefined` → defaulting to TTS-on. Fixed by renaming to `tts_enabled`.
- **SPEAKING animation stuck** — TTS-disabled path was emitting `chat-audio-done`
  immediately, killing the animation before typewriter finished. Removed the premature
  emit; animation now stops via reactive statement when streaming ends.
- **Link text erased by sanitizer** — `sanitizeHtml` set `link.innerHTML = ''`, making
  all anchor labels invisible. Fixed to preserve `textContent` while stripping unsafe
  attributes.
- **Dead frontend listener** — removed unused `fn-key-double-click` listener (Rust
  never emits this event).
- **Unused CSS warnings** — `.message-text a` selectors scoped with `:global()` to
  suppress false-positive build warnings for `{@html}` rendered content.
- **Settings UI stale hotkey name** — display name now always derived from
  `hotkey_code` via Rust's `update_hotkey()` return value.
- **False recording trigger when changing hotkey** — captured modifier key release
  events correctly identified via state tracker.
