# Pocket Agent v0.2.4

**Release Date**: 2026-05-16

Focused reliability release for macOS hotkeys and voice interruption. This version closes the local-vs-SSH hotkey gap, fixes modifier capture edge cases, and makes TTS interruption immediate when the user starts talking again.

## Highlights

### Local + SSH hotkey parity
- **fn/globe normalization** — treats macOS `FLAGS_CHANGED` keycode `63` and `KEY_DOWN` keycode `179` as the same canonical `fn` hotkey.
- **Local launch fix** — local terminal launches can now use `fn` from the modifier-event path instead of depending on `KEY_DOWN` only.
- **SSH duplicate suppression** — when SSH-launched sessions emit both `FLAGS_CHANGED` and a duplicate `KEY_DOWN` for one modifier press, Pocket Agent swallows the duplicate so recording toggles exactly once.

### Modifier capture stability
- **First-press-after-capture fix** — capturing a modifier hotkey no longer poisons the next real press with the release event from capture mode.
- **Capture reset hygiene** — capture state now clears one-shot suppression flags before starting a new key capture.

### Event tap resilience
- **Tap auto-recovery** — if macOS disables the listen-only CGEventTap because of timeout or user input conditions, Pocket Agent re-enables it automatically instead of silently losing the global hotkey.

### Immediate TTS interruption
- **Synchronous stop path** — hotkey start now interrupts the active `rodio::Sink` immediately instead of waiting for the playback thread to drain its queue.
- **Generation-aware queue invalidation** — old queued audio jobs are invalidated on stop so stale TTS does not resume after a new recording begins.
- **Safe queue accounting** — queue release logic now avoids underflow after interruption/reset, preserving backpressure correctness.

## FAQ summary
- **Local `tauri dev` hotkey dead?** Grant Input Monitoring to the terminal app that launched PA.
- **SSH run double-triggers on one press?** Fixed in v0.2.4 by suppressing the duplicate modifier `KEY_DOWN`.
- **First press after changing to `fn`/Shift does nothing?** Fixed in v0.2.4 by consuming the capture-release event before normal matching.
- **PA keeps speaking after hotkey press?** Fixed in v0.2.4 with synchronous `sink.stop()` interruption.
