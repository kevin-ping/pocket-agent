# Pocket Agent v0.2.0

**Release Date**: 2026-05-04

Performance overhaul — 6x faster response time (from ~3 min to ~30 sec), zero-latency recording, and daily session rotation.

## Breaking Changes

- **SESSION_ID removed from .env** — session ID is now hardcoded as `pocket-agent` with automatic daily date suffix (`pocket-agent-YYYY-MM-DD`). Remove `SESSION_ID` from your `.env` file.

## New Features

### Stream Daemon (Zero-Latency Recording)
- **Pre-warmed audio stream** — cpal Stream is built at startup and kept alive in a background daemon thread. Pressing the hotkey only flips `recording=true` (~11ms), eliminating the 200-500ms `AudioUnitInitialize` delay from the previous architecture.
- **No more truncated speech** — first words are no longer cut off. Audio capture begins almost instantly on keypress.
- **Automatic stream rebuild** — after each recording stops, the daemon immediately pre-builds the next stream for the following recording.

### Daily Session Rotation
- **Date-based session IDs** — format `pocket-agent-YYYY-MM-DD`, automatically rotates at midnight. No configuration needed.
- **Daily context compression** — a cron job at 00:00 compresses yesterday's session into a ~500 word summary, stored in `~/.hermes/pa-summaries/`.
- **Context injection** — PA loads yesterday's summary as ephemeral context on each new day, maintaining conversation continuity without bloating the session.

### STT Progress Logging
- **Real-time transcription progress** — STT phase now prints progress to terminal (`loading model`, `transcribing`, `done`). Previously a silent ~20s black box.

## Improvements

### Token Efficiency
- **Voice hint as system message** — TTS instructions are now sent as `role: system`, which Hermes gateway treats as ephemeral (injected per request, not stored in DB). Previously duplicated ~812 chars per user message in session history.
- **Result**: per-message token cost dropped from ~1861 to ~120 (comparable to Telegram).

### Language Detection
- **Word-level English counting** — `detect_language()` now counts English by word (space-separated tokens ≥2 letters) instead of by individual letters. `"测试 response"` correctly identifies as Chinese (zh=2, en=1) instead of English (zh=2, en=8).
- **Auto language following** — in non-forced mode, LLM is instructed to respond in the same language as the user's input, preventing context drift after mixed-language conversations.

### Race Condition Fixes
- **Stop handles start-voice race** — `stop_voice_recording` now retries up to 500ms when handle is not yet set, preventing the case where hotkey-up arrives before `start_voice_recording` completes.
- **Stream rebuild gap** — `pre_start()` no longer silently drops Start commands when the stream is being rebuilt; commands queue in the channel and are processed immediately after rebuild.

## Removed

- **Debug footer** — removed the `[DEBUG: session_messages=N, estimated_input_tokens=~T]` instruction from voice hints that was appended to every LLM response.

## Technical Details

### voice_hint → role: system
Hermes `api_server.py` (L888-901) extracts messages with `role: "system"` as `ephemeral_system_prompt` — injected into every LLM call but never stored in the session's `messages` table. This is the same behavior as OpenAI's Chat Completions API where `system` is transient and `user/assistant` is persistent.

### Stream Daemon Architecture (`record.rs`)
```
Daemon Thread (lives forever):
  loop {
    Phase 1: build_input_stream (~200ms)  ← expensive, done in background
    Phase 2: wait for Start command       ← blocks on channel
    Phase 3: stream.play() + wait for Stop ← ~11ms to start capture
    → drop stream, loop back to Phase 1
  }

Main Thread:
  pre_start()  → send Start to daemon (non-blocking, <1ms)
  stop_recording() → send Stop to daemon → finalize WAV
```

### Daily Compression Cron
- Script: `~/.hermes/scripts/pa_daily_compress.py`
- Reads yesterday's messages from `~/.hermes/state.db`, compresses via LLM, writes to `~/.hermes/pa-summaries/YYYY-MM-DD.md`
- PA reads this file on startup and injects as ephemeral system context
