# Pocket Agent v0.2.2

**Release Date**: 2026-05-07

LLM thinking phase visualization — see what your agent is doing during those long waits. Also fixes session duplication and remembers where you left your window.

## New Features

### LLM Thinking Phase Display
- **Real-time thinking steps** — during the LLM thinking/tool-calling phase, PA now shows intermediate steps in the dialog box instead of a blank cursor blink.
- **Reasoning visualization** — LLM reasoning text (e.g., DeepSeek R1 thinking process) appears as `🤔 ...` lines, buffered and batched for readability (flushed at 60 chars or paragraph breaks).
- **Tool call indicators** — when Hermes backend calls a tool, a `🔧 查询 yahoo price...` step appears immediately. Tool names are cleaned up (`mcp_tradingview_yahoo_price` → `yahoo price`).
- **Fade-in animation** — steps appear with a smooth fade-in so the user can track progression.
- **Auto-clear on response** — once the LLM finishes and TTS is ready, all intermediate steps are cleared and the normal typewriter + voice playback begins.

### Window Position Memory
- **Remembers where you left it** — PA now saves its window position when you drag it, and restores it on next launch.
- **Debounced save on drag** — position is saved 500ms after you stop dragging (via `onMoved` event), no polling.
- **Backup save on close** — also saves when the window is closed, just in case.
- **First launch fallback** — if no saved position exists, PA opens at the system default position (top-left corner).

## Bug Fixes

- **Session key mismatch causing session duplication** — when `API_AGENT=xingyin` was set, PA only sent `x-openclaw-session-key` header. Hermes doesn't recognize this header and created a new session for every request. Fixed by **always sending `X-Hermes-Session-Id`** first, then additionally sending `x-openclaw-session-key` for OpenClaw compatibility.
- **Tauri 2 API incompatibility** — `appWindow.currentMonitor()` is not a method in Tauri 2. Replaced with `appWindow.scaleFactor()` which is the correct API.

## Improvements

- **Tool progress format compatibility** — now supports both Hermes' custom SSE events (`event: hermes.tool.progress`) and standard OpenAI `delta.tool_calls` format.
- **Reasoning text batching** — tiny SSE reasoning chunks are buffered in Rust and only emitted to the frontend when meaningful (60+ chars or newline encountered), preventing UI flooding.

## Technical Details

### Thinking Phase Flow

```
User sends message → startStream()
  ↓
🤔 正在思考...                    ← initial step
🤔 让我分析一下用户的需求...       ← reasoning updates (buffered)
🔧 查询 yahoo price...            ← tool call appears immediately
🤔 正在综合结果...                ← more reasoning
  ↓
LLM done → TTS ready → clearThinkingSteps()
  ↓
Typewriter + voice playback begins
```

### SSE Event Parsing

Hermes backend emits tool progress via custom SSE events:

```
event: hermes.tool.progress
data: {"tool": "mcp_tradingview_yahoo_price", "status": "running"}
```

Standard OpenAI format (`delta.tool_calls`) is also supported as fallback.

### Session Headers (Fixed)

```
Always sent:
  X-Hermes-Session-Id: pocket-agent-2026-05-07

Additionally sent when API_AGENT is set:
  x-openclaw-session-key: agent:xingyin:pocket-agent-2026-05-07
```
